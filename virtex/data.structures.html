virtex.data.structures¶

This module contains a bunch of dict-like data structures used by datasets in virtex.data.datasets for returning instances and batches of training (or inference) data.

These classes are thin wrappers over native python dicts for two main purposes:

  1. Better readability, type-hint annotations and stronger static type checking.

  2. Each of these classes implement __slots__: a trick to significantly reduce memory overhead in creating hundreds of new dict objects (every iteration) if the name of dict keys remain fixed (and unchanged).

Base Structures¶

class virtex.data.structures.Instance[source]¶

Bases: dict

Base class for representing a single instance: a dict of key value pairs. Keys are assumed to be str, and values are assumed to be torch.Tensor.

This class can be instantiated with **kwargs just like dict().

to(self, *args, **kwargs) → ’Instance’[source]¶

Defines the logic to move the whole instance across different :class:`torch.dtype`s and :class:`torch.device`s. Default implementation shifts all tensor-like objects to device.

Note

This method is used internally by NVIDIA Apex for casting the instance to FP16 (half) precision – this method casts floats to half; while keeping integers, booleans and other data types unchanged.

pin_memory(self)[source]¶

Pin GPU memory; only used internally by PyTorch dataloaders.

class virtex.data.structures.Batch[source]¶

Bases: dict

Base class for representing a single batch. It is created with a list of instances; and used by collate_fn of virtex.data.datasets. This is exactly same as Instance in terms of behavior and functionality.
